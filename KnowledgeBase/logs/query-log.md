## 📝 查询记录格式

每条记录包含：

```markdown
### [查询类型] 查询内容

- **时间**: YYYY-MM-DD HH:MM:SS
- **用户**: [用户标识]
- **来源页面**: [如果有上下文]
- **引用页面**: [回答中引用的 Wiki 页面]
- **满意度**: ⭐⭐⭐⭐⭐
- **备注**: [额外信息]
```

### 示例

```markdown
### 知识查询 我们的认证模块是如何设计的？

- **时间**: 2026-04-17 10:30:00
- **用户**: team-member-1
- **引用页面**: 
  - [Entity-AuthenticationModule](../entities/Entity-AuthenticationModule.md)
  - [Concept-CQRS](../concepts/Concept-CQRS.md)
- **满意度**: ⭐⭐⭐⭐⭐
- **备注**: 回答完整，链接清晰
```

## 🔔 异常查询

### 模糊查询

```
用户提问过于宽泛，无法准确回答
→ 记录并建议具体化问题
```

### 超出范围

```
查询与项目无关的内容
→ 记录并说明范围限制
```

### 敏感信息

```
查询涉及权限限制的内容
→ 记录并检查访问权限
```

## 📈 查询优化

### 基于日志的改进

1. **识别热门话题** → 创建综合论述
2. **发现知识缺口** → 补充缺失页面
3. **优化响应时间** → 预加载常用页面
4. **改进答案质量** → 更新过时信息

### 自动建议

LLM 会根据查询日志主动建议：

```
我注意到以下模式：
1. 本周 3 次查询关于 "事件溯源"
2. 相关信息分散在 4 个文档中
3. 建议创建 Synthesis-Event-Sourcing

是否要我创建？
```
