## 🎯 这是什么？

这是一个基于 Karpathy LLM Wiki 理念构建的**智能知识库系统**。

### 核心理念

与传统 RAG 系统不同，这个 Wiki 系统：

- ✅ **持久化知识积累**：不是每次查询都重新检索，而是持续构建知识体系
- ✅ **增量式更新**：添加新文档时，LLM 自动提取、整合、更新现有知识
- ✅ **结构化关联**：文档间自动建立交叉引用，形成知识网络
- ✅ **合成与演进**：从原始文档 → 摘要 → 实体/概念 → 综合论述，层层提炼

### 你能做什么？

- 📝 **添加文档**：放入需求、设计、会议纪要等
- 🤖 **让 LLM 处理**：自动创建摘要、实体页面、概念页面
- 🔍 **查询知识**：提问获取答案，追溯信息来源
- 📊 **浏览知识**：通过索引和图谱探索知识网络

## 🚀 快速开始

### 1. 了解目录结构

```
KnowledgeBase/
├── raw-docs/           # 📄 你放原始文档的地方
│   ├── requirements/   # 需求文档
│   ├── design/         # 设计文档
│   ├── meetings/       # 会议纪要
│   └── ...
│
├── wiki/               # 🧠 LLM 维护的知识库
│   ├── entities/       # 实体页面（人、技术、产品等）
│   ├── concepts/       # 概念页面（模式、原则等）
│   ├── summaries/      # 文档摘要
│   └── synthesis/      # 综合论述
│
├── indexes/            # 📑 索引（快速导航）
├── logs/               # 📜 日志（自动记录）
├── templates/          # 📋 模板（LLM 使用）
└── schema/             # ⚙️ 配置（系统规则）
```

### 2. 添加你的第一篇文档

#### 步骤 1：准备文档

创建一个 Markdown 文件，例如：

```markdown
# SBLD 核心功能需求

## 背景
[描述背景]

## 目标
[列出目标]

## 功能需求
1. [功能 1]
2. [功能 2]
```

#### 步骤 2：放入正确位置

保存到：`KnowledgeBase/raw-docs/requirements/20260417-core-features.md`

#### 步骤 3：告诉 LLM

```
"请处理这个新文档：KnowledgeBase/raw-docs/requirements/20260417-core-features.md"
```

#### 步骤 4：查看结果

LLM 会创建：
- ✅ 摘要页面：`wiki/summaries/Summary-core-features.md`
- ✅ 实体页面：`wiki/entities/Entity-核心模块.md`
- ✅ 概念页面：`wiki/concepts/Concept-相关概念.md`
- ✅ 更新索引：`indexes/*`

## 📖 使用场景

### 场景 1：添加需求文档

```
1. 写需求文档 → raw-docs/requirements/
2. 告诉 LLM："请处理这个需求文档"
3. LLM 创建摘要、提取实体和概念
4. 审查 LLM 提取的信息
5. 完成！
```

### 场景 2：查询知识

```
问："我们的认证模块是如何设计的？"

LLM 会：
1. 检索相关实体页面
2. 检索相关概念页面
3. 检索相关综合论述
4. 综合回答，并附上链接
```

### 场景 3：添加会议纪要

```
1. 整理会议纪要 → raw-docs/meetings/
2. 告诉 LLM："请处理这个会议纪要"
3. LLM 提取：
   - 关键决策
   - 待办事项
   - 相关实体更新
4. 自动更新相关 Wiki 页面
```

### 场景 4：知识维护

```
问："最近有哪些文档需要更新？"

LLM 会：
1. 检查过期的 Wiki 页面
2. 识别孤立的实体/概念
3. 发现知识缺口
4. 生成维护报告
```

## 🎓 核心概念

### 三层架构

```
原始文档 (raw-docs/)
    ↓ 读取
Wiki 知识 (wiki/)
    ↓ 综合
综合论述 (synthesis/)
```

### 四种页面类型

| 类型 | 用途 | 示例 |
|------|------|------|
| **实体** | 描述具体事物 | Entity-AuthenticationModule |
| **概念** | 描述抽象理念 | Concept-CQRS |
| **摘要** | 总结单篇文档 | Summary-core-features |
| **综合** | 跨文档综合分析 | Synthesis-Auth-Patterns |

### 交叉引用

所有页面通过链接相互关联：

```markdown
[Entity-AuthenticationModule](entities/Entity-AuthenticationModule.md)
[Concept-CQRS](concepts/Concept-CQRS.md)
[Summary-core-features](summaries/Summary-core-features.md)
```

## ❓ 常见问题

### Q: 我可以直接编辑 Wiki 页面吗？

**A**: 不推荐。让 LLM 维护 Wiki，你在对话中提出修改建议。这样可以保持：
- 版本一致性
- 交叉引用完整性
- 变更日志可追溯

### Q: 文档格式有要求吗？

**A**: 推荐使用 Markdown，因为：
- LLM 解析最准确
- 易于阅读和编写
- 支持内部链接
- 版本控制友好

### Q: 如何处理大量文档？

**A**: 分批处理：
1. 先处理重要的 10 篇文档
2. 审查结果，调整配置
3. 批量处理剩余文档
4. 定期维护健康度

### Q: Wiki 页面太多怎么办？

**A**: 系统会自动分片：
- 超过 50 个文件 → 创建子目录
- 超过 1000 行索引 → 分区索引
- 复杂主题 → 专题 Wiki

### Q: 如何备份？

**A**: 推荐方式：
- 使用 Git 版本控制
- 定期导出到云存储
- 保留变更日志
